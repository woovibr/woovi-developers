## Registro de conta pendente

Os webhooks de `ACCOUNT_REGISTER_PENDING` enviam objetos com os seguintes campos no seu payload:

- `accountRegister`: Detalhes do registro de conta
  - `officialName`: Nome oficial/razão social da conta
  - `taxID`: Objeto contendo o documento de identificação
    - `taxID`: Número do CPF ou CNPJ
    - `type`: Tipo do taxID (`BR:CPF` ou `BR:CNPJ`)
  - `status`: Status do registro (sempre `PENDING` para este evento)
  - `requestDocuments`: Lista de documentos solicitados para análise
  - `requestReason`: Comunicado sobre as pendências
  - `requestDocumentsRepresentatives`: Lista de documentos solicitados para os representantes legais

## Quando é enviado

Este evento é enviado quando um registro de subconta está em análise pelo compliance. Isso pode ocorrer quando:

- Uma nova conta é criada, entra em processo de análise mas algun documento é rejeitado e tem que ser reenviado
- Documentos adicionais são solicitados para a conta

## Exemplo

```json
{
  "event": "ACCOUNT_REGISTER_PENDING",
  "accountRegister": {
    "officialName": "Test Company",
    "taxID": {
      "taxID": "12345678901234",
      "type": "BR:CNPJ"
    },
    "status": "PENDING",
    "requestDocuments": [],
    "requestReason": "",
    "requestDocumentsRepresentatives": []
  }
}
```

## Exemplo com documentos solicitados

```json
{
  "event": "ACCOUNT_REGISTER_PENDING",
  "accountRegister": {
    "officialName": "Empresa Exemplo LTDA",
    "taxID": {
      "taxID": "12345678000199",
      "type": "BR:CNPJ"
    },
    "status": "PENDING",
    "requestDocuments": ["SOCIAL_CONTRACT", "ADDRESS_PROOF"],
    "requestReason": "Faltou contrato social e comprovante de endereço",
    "requestDocumentsRepresentatives": [
      {
        "taxId": {
          "taxID": "12345678901",
          "type": "BR:CPF"
        },
        "requestDocuments": ["RG", "ADDRESS_PROOF"]
      }
    ]
  }
}
```
